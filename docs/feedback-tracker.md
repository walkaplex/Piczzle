# Piczzle Feedback Tracker

Use this file to collect tester feedback during friends-and-family testing.

## Intake Rules

- Add one row per issue, question, or useful comment.
- Keep raw tester wording in `Tester note` when possible.
- Use `Needs triage` until we decide whether it is a bug, polish item, or expected behavior.
- Do not add private photos, personal phone numbers, or unrelated personal details.
- If a tester sends screenshots, save them outside Git unless they are safe to share publicly.

## Status Values

- `Needs triage`
- `Accepted`
- `Needs repro`
- `Fixed`
- `Won't fix for beta`
- `Known limitation`

## Severity Values

- `High`: blocks creating, solving, saving, sharing, or opening the app.
- `Medium`: confusing or broken, but a tester can continue.
- `Low`: visual polish, wording, minor layout issue, or nice-to-have.

## Feedback Areas

- `Invite`: invite pages, install/open instructions, link previews.
- `Setup`: choose photo, demo selection, camera/gallery.
- `Crop`: framing, zooming, preview.
- `Play`: board, pieces, hint, restart, solve, touch behavior.
- `Completion`: solved puzzle popup, view puzzle, save image, feedback.
- `Share`: create link, copy link, share sheet, open shared puzzle.
- `Save`: save image behavior on web, Android, or iOS Safari.
- `Performance`: slow load, lag, freezing, battery, memory.
- `Visual`: cut-off content, spacing, contrast, safe area, polish.
